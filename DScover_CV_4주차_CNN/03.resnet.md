## 1. ILSVRC와 깊이의 변화

ImageNet Large Scale Visual Recognition Challenge(ILSVRC) 우승 모델을 연도별로 보면, **오류율은 내려가고 층 수(depth)는 커지는** 흐름이 뚜렷하다.

| 대회 / 모델 | 오류율 (%) | 깊이 |
|-------------|------------|------|
| ILSVRC’10 | 28.2 | shallow |
| ILSVRC’11 | 25.8 | shallow |
| ILSVRC’12 (AlexNet) | 16.4 | 8 layers |
| ILSVRC’13 (ZFNet) | 11.7 | 8 layers |
| ILSVRC’14 (VGG) | 7.3 | 19 layers |
| ILSVRC’14 (GoogLeNet) | 6.7 | 22 layers |
| **ILSVRC’15 (ResNet)** | **3.57** | **152 layers** |

**ResNet**은 기존 모델 대비 **층 수를 크게 늘리면서** 성능 향상을 이끈 구조로 기억하면 된다.

(참고 읽기: [7 CNN Architectures (2010–2015)](https://medium.com/analytics-vidhya/7-cnn-architectures-evolved-from-2010-2015-fd5869bd744e))

## 2. 논문이 말하는 핵심

- **Degradation problem**을 완화하기 위해 **residual block(잔차 블록)** 구조를 제안했다.
- **실험적으로** “깊어져도 학습이 망가지지 않게” 만드는 효과를 보였다.

논문 Figure 1(CIFAR-10, **plain** 네트워크): **56층**이 **20층**보다 **training error·test error 모두 더 크다**.  
→ 단순히 깊게 쌓은 plain 네트워크에서는 **오버피팅만의 문제가 아니다**(학습 자체가 어려움).

---

## 3. Degradation Problem

**관찰**

- 층을 깊게 할수록 성능이 **나빠지는** 현상이 있다.
- 이때 **training error도 함께 나빠지므로**, 전형적인 **overfitting만으로 설명되지 않는다**.

**가설(논의)**

- 깊은 층은 **최적화(optimization)가 어렵다**.
- 이런 현상을 **degradation problem**이라고 부른다.

---

## 4. ResNet의 아이디어: Residual Learning

원하는 매핑을 $H(x)$라 하고, 쌓인 비선형 층이 학습할 대상을 **잔차**로 두자:

$$
F(x) := H(x) - x,\quad \text{출력은 } F(x) + x \text{ 로 재구성}
$$

**가설:** 원래 매핑 $H(x)$ 전체보다, **잔차 $F(x)$** 가 **최적화하기 쉽다**.

블록 단위 식(논문 식 (1)과 동일한 형태):

$$
y = \mathcal{F}(x, \{W_i\}) + x
$$

- $\mathcal{F}$: weight layer들을 통과한 **잔차 경로**
- $x$: **shortcut(identity)** 로 더해지는 입력
- 이후 **ReLU** 등 활성화가 이어짐 (Figure 2: *Residual learning: a building block*)

### VGG 대비 직관

“만약 얕은 구조(예: VGG-19)가 이미 충분히 좋다면, 더 깊은 네트워크의 **추가 층은 사실상 identity에 가깝게** 동작할 수 있어야 한다.”  
Residual connection은 그런 **identity 근사**를 학습하기 쉽게 만든다는 직관과 맞닿아 있다.

**34-layer plain vs 34-layer residual:** 레이어 구성은 같게 두고, residual 쪽만 **shortcut**(실선=차원 맞는 identity, 점선=차원이 바뀌는 지점의 projection)을 추가한 형태로 이해하면 된다.

### Figure 4 (학습 곡선, ImageNet)

- **Plain 18 vs 34:** 34층 plain이 **더 높은 error** → degradation.
- **ResNet-18 vs ResNet-34:** 34층 ResNet이 **더 낮은 error** → residual이 의도대로 동작.

### Table 2 (Top-1 error %, 10-crop, ImageNet validation)

| 깊이 | Plain | ResNet |
|------|-------|--------|
| 18 | 27.94 | 27.88 |
| 34 | 28.54 | **25.03** |

→ 같은 파라미터 증가 없이(plain 대비) **residual만으로** 깊이를 늘렸을 때 이득이 난다는 메시지.

## 6. Identity vs Projection Shortcut (A / B / C)

차원이 맞지 않을 때 shortcut을 어떻게 할지에 대한 선택지:

| 옵션 | 요약 |
|------|------|
| **(A)** | 차원 늘릴 때 **zero-padding** 등으로 맞춤. **파라미터 없음**. |
| **(B)** | 차원이 바뀌는 지점만 **projection**(예: $1\times1$ conv), 나머지는 **identity**. |
| **(C)** | **모든** shortcut을 projection으로 처리. |

**관찰:** A/B/C 사이 성능 차이는 **작다** → projection이 degradation 해결의 **필수 조건은 아니다**.

**논문 선택:** **(C)는 이후 실험에서 쓰지 않음** — 정확도 이득 대비 **연산·메모리·모델 크기** 부담이 크다.  
**ResNet-50 / 101 / 152**는 **옵션 B**를 사용한다고 논문에서 명시한다.

### Table 3 일부 (ImageNet validation, top-1 / top-5 %)

| 모델 | top-1 | top-5 |
|------|-------|-------|
| Plain-34 | 28.54 | — |
| ResNet-34 A | 25.03 | 7.76 |
| ResNet-34 B | 24.52 | 7.46 |
| ResNet-34 C | 24.19 | 7.40 |
| ResNet-50 | 22.85 | 6.71 |
| ResNet-101 | 21.75 | 6.05 |
| ResNet-152 | 21.43 | 5.71 |

## 7. Bottleneck 블록 (ResNet-50 / 101 / 152)

**목적:** $1\times1$ conv로 **채널을 줄였다가(병목)** $3\times3$로 공간 처리 후, 다시 $1\times1$로 **복원**해 **연산량을 줄인다**.

- **ResNet-34:** 기본 블록은 대개 **$3\times3$ 두 개** 스택.
- **ResNet-50/101/152:** **$1\times1 \rightarrow 3\times3 \rightarrow 1\times1$** bottleneck (논문 Figure 5).

---

## 8. 다양한 ResNet 구성 (Table 1 요약)

같은 스케줄로 **conv2_x ~ conv5_x**에 블록을 쌓되, **18·34**는 basic(2개의 $3\times3$), **50+**는 bottleneck(3개 conv 스택)을 쓴다.

**FLOPs (ImageNet, 논문 Table 1)**

| 모델 | FLOPs |
|------|--------|
| ResNet-18 | \(1.8 \times 10^9\) |
| ResNet-34 | \(3.6 \times 10^9\) |
| ResNet-50 | \(3.8 \times 10^9\) |
| ResNet-101 | \(7.6 \times 10^9\) |
| ResNet-152 | \(11.3 \times 10^9\) |

**다운샘플링:** `conv3_1`, `conv4_1`, `conv5_1` 첫 블록에서 stride 2 등으로 수행한다고 논문에 정리되어 있다.

## 10. CIFAR-10 실험 (Figure 6 요지)

- **Plain:** 층 수를 늘리면(20→56) error가 **악화** — degradation.
- **ResNet:** 층을 깊게 해도(20→110) error가 **개선**되는 쪽으로 동작.
- **110 vs 1202:** 둘 다 training error는 매우 낮은데, **1202층**이 test에서 **110보다 나쁨** → 데이터·모델 규모 관점에서 **과도한 깊이는 overfitting** 등 새 문제를 부를 수 있음.

## 11. Object Detection에의 응용

Faster R-CNN에서 **backbone을 VGG-16 → ResNet-101**로 바꿀 때 **mAP가 일관되게 상승**한다는 식의 비교가 논문 Table 7·8에 제시된다 (PASCAL VOC, COCO 등).

---

## 12. ResNet의 의의

- **Identity에 가까운 shortcut**으로 **최적화를 쉽게** 해 주어, **층 수를 크게 늘리는** 설계가 현실화되었다.
- 이후 **100층 이상** CNN을 당연한 선택지로 만든 **전환점**이 된 아키텍처다.

---

## 한 줄 정리

**ResNet**은 $y = \mathcal{F}(x)+x$ 형태의 **잔차 학습 + shortcut**으로 degradation을 완화하고, **매우 깊은 네트워크**를 ImageNet·다운스트림 태스크에서 **검증된 형태**로 굳혔다.
