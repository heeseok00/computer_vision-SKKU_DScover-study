# `tf.train.CheckpointManager` API로 파라미터 저장·불러오기

> 예제 코드: [`01.mnist_classification_using_cnn_with_tfsaver_v2_keras.py`](./01.mnist_classification_using_cnn_with_tfsaver_v2_keras.py)

---

## 3. 파라미터 저장과 복원 (Checkpoint)

### 파라미터 저장이 필요한 이유

- 실제 딥러닝 문제는 학습 시간이 매우 길어 여러 epoch 반복이 필요함
- 학습 도중 문제가 발생할 수 있기 때문에 **중간 결과 저장**이 필요함
- 저장된 파라미터를 활용하면:
  - 이어서 학습 가능
  - 가장 성능 좋은 모델 사용 가능
  - 재학습 시간 절약 가능

TensorFlow에서는 **`tf.train.CheckpointManager` API**를 제공한다.

---

## Checkpoint 구조 개념

Checkpoint에 둘 수 있는 구성 요소(예시):

- **Model** (가중치)
- **Optimizer** 상태
- **Training step** 정보

즉, 필요에 따라 **학습 상태 전체**를 저장할 수 있다.

이 예제 스크립트에서는 **`model`**과 **`step`**만 `Checkpoint`에 넣어 저장한다.

---

## 파라미터 저장 과정

### 1. Checkpoint 객체 생성

저장할 대상을 지정한다. (예: `model`, `step`)

```python
ckpt = tf.train.Checkpoint(
    step=tf.Variable(0),
    model=CNN_model
)
```

---

### 2. CheckpointManager 생성

저장 **디렉터리**와 유지할 체크포인트 **개수**를 설정한다.

```python
ckpt_manager = tf.train.CheckpointManager(
    ckpt,
    directory=SAVER_DIR,
    max_to_keep=5
)
```

- **`max_to_keep`**: 최근 몇 개의 checkpoint를 유지할지 설정

예제에서는 `SAVER_DIR = "./model"` 을 사용한다.

---

### 3. 파라미터 저장

학습 루프 안에서 필요할 때 저장한다.

```python
ckpt_manager.save(checkpoint_number=ckpt.step)
```

보통은 **epoch이 끝날 때** 또는 **일정 step마다** 저장한다.  
예제는 **100 step마다** 저장한다.

---

### 4. step 증가

학습 iteration을 기록한다.

```python
ckpt.step.assign_add(1)
```

즉, **현재 학습 진행 상태**를 숫자로 남긴다.

---

## 파라미터 불러오기 과정

### 1. 최신 checkpoint 찾기

저장된 checkpoint 중 **가장 최근** 경로를 가져온다.

```python
latest_ckpt = tf.train.latest_checkpoint(SAVER_DIR)
```

---

### 2. 파라미터 복원

저장된 가중치 등을 불러온다.

```python
ckpt.restore(latest_ckpt)
```

결과: **학습 상태가 저장 시점으로 복구**된다.

예제에서는 복원에 성공하면 테스트 정확도를 출력한 뒤 `exit()`로 종료한다.

---

## 핵심 정리

**CheckpointManager**는 학습 중 **모델(및 지정한 상태) 저장·관리**를 돕는 도구다.

**저장 흐름**

1. `Checkpoint` 생성  
2. `CheckpointManager` 생성  
3. `save` 호출  
4. `step` 증가  

**복원 흐름**

1. `latest_checkpoint`  
2. `restore`  

---

## 한 줄 핵심

**CheckpointManager** → 딥러닝 학습 **중간 상태를 저장**하고, **이어서 학습**하거나 **저장된 모델로 평가**할 수 있게 하는 기능이다.
