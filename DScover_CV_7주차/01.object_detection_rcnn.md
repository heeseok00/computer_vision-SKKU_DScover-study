## Part B — Object Detector 분류: One-stage vs Two-stage

![One-stage vs Two-stage Detector](images/02_one_stage_vs_two_stage.png)

Object Detector는 크게 두 가지로 나뉜다.

- **Two-stage Detector**: Region Proposal → Classification의 두 단계로 나뉨
  - 대표 모델: R-CNN, Fast R-CNN, **Faster R-CNN** (강의 주요 포커스)
  - 정확도가 높지만 속도가 느림
- **One-stage Detector**: 두 단계를 하나로 합쳐 한 번에 검출
  - 대표 모델: YOLO, SSD
  - 빠르지만 상대적으로 정확도가 낮을 수 있음

## Part D — R-CNN 개요

![R-CNN 개요 (Regions with CNN features)](images/04_rcnn_overview.png)

- **논문**: Girshick, Ross, et al. *"Rich feature hierarchies for accurate object detection and semantic segmentation."* CVPR 2014.
- **논문 링크**: [arxiv.org/pdf/1311.2524](https://arxiv.org/pdf/1311.2524.pdf)

2012년 AlexNet이 Image Classification 분야에서 기존 모델들을 압도하는 성능을 보여줬다.  
R-CNN은 그보다 한 단계 더 어려운 **Object Detection** 분야에서도 딥러닝이 통한다는 것을 처음으로 보여준 연구로, 이후 수많은 Detection 모델 연구의 발판이 됐다.

**R-CNN 추론 흐름 (4단계)**

1. Input image 입력
2. Region Proposal 추출 (~2,000개)
3. 각 region에서 CNN feature 계산
4. 각 region을 분류 (Classify regions)

## Part E — R-CNN Architecture

![R-CNN Architecture 상세](images/05_rcnn_architecture.png)

- 이미지에서 약 **2,000개의 region proposal**을 추출
- 각 region을 **227×227**로 warp(crop+scale)하여 CNN에 입력
- CNN 출력으로 **linear SVM**을 통해 클래스 분류
- 병렬 처리가 아닌 각 region을 **순차적으로** CNN에 통과시키므로 속도가 느림

---

## Part F — Selective Search를 이용한 Region Proposal

![Selective Search Region Proposal](images/06_selective_search.png)

- **Selective Search**: 이미지의 Color, Size, Fill, Texture 등의 특성을 기반으로 bounding box 후보군을 러프하게 제안하는 알고리즘
- 딥러닝이 아닌 **전통적인 컴퓨터 비전 방법**으로 후보 영역을 생성
- 이 단계가 병목이 되어 Faster R-CNN에서는 **RPN(Region Proposal Network)** 으로 대체됨

## Part G — R-CNN 추론 단계 상세

### G-1. Test Step 2 — Region Crop & Scale

![R-CNN Test Step 2](images/07_rcnn_test_step2.png)

- Selective Search로 뽑은 region을 **crop** 후 **227×227로 anisotropic scaling**
- 각각을 독립적으로 CNN에 입력

### G-2. Test Step 3 — Feature Extraction & Classification

![R-CNN Test Step 3](images/08_rcnn_test_step3.png)

- CNN을 통해 **4,096-dimensional fc7 feature vector** 추출
- 이를 **linear classifier (SVM 또는 softmax)** 에 넣어 클래스 분류

### G-3. Test Step 4 — Bounding Box Regression

![R-CNN Test Step 4 (Object proposal refinement)](images/09_rcnn_test_step4_bbox.png)

- Selective Search로 뽑은 원래 proposal은 위치가 정확하지 않을 수 있음
- CNN feature를 기반으로 **linear regression**을 수행해 bounding box 위치를 보정
- 이를 **Bounding-box regression**이라고 함

## Part H — R-CNN Bounding Box Linear Regression

![Bounding Box Linear Regression (Ridge Regression)](images/10_rcnn_bbox_regression_formula.png)

$$\mathbf{w}_\star = \underset{\hat{\mathbf{w}}}{\operatorname{argmin}} \sum_i (t^i_\star - \hat{\mathbf{w}}^\top \phi_5(P^i))^2 + \lambda \|\hat{\mathbf{w}}_\star\|^2$$

- **Ridge Regression** 형태: L2 regularization(`λ‖w‖²`)을 사용
- `φ₅(Pⁱ)`: proposal Pⁱ에서 뽑은 CNN feature(pool5)
- `tⁱ`: ground-truth box까지의 offset (dx, dy, dw, dh)
- regression target은 **normalized coordinates**로 표현
  - `(0, 0, 0, 0)` → proposal이 정확함
  - `(0.25, 0, 0, 0)` → proposal이 너무 왼쪽에 치우침
  - `(0, 0, -0.125, 0)` → proposal이 너무 넓음

## Part I — R-CNN 학습 과정

### I-1. Training Step 1 — Supervised Pre-training

![R-CNN Training Step 1](images/11_rcnn_train_step1.png)

- **ILSVRC 1,000-class image classification** 태스크로 CNN(AlexNet 계열)을 사전 학습
- 1.2M 이미지, 분류 태스크 기반의 **supervised pre-training**

### I-2. Training Step 2 — Domain-specific Fine-tuning

![R-CNN Training Step 2](images/12_rcnn_train_step2.png)

- ILSVRC 분류로 학습된 CNN을 **PASCAL VOC detection** 태스크에 맞게 fine-tuning
- target task: PASCAL VOC (~25k object labels)
- Classification head를 detection 용도로 교체

### I-3. Training Mini-Batch 구성 (CNN Fine-tuning)

![Training Mini-Batch](images/13_rcnn_minibatch.png)

- IoU ≥ 0.5: ground-truth box와 **0.5 이상 겹치는** region → **positive** (해당 클래스)
- 나머지 → **negative** (배경)
- Mini-batch 구성: **32 positive + 96 background = 128**
- positive 비율이 극히 낮기 때문에 positive를 의도적으로 많이 샘플링 (biased sampling)

### I-4. Training Step 3 — Detection SVM 학습

![R-CNN Training Step 3](images/14_rcnn_train_step3.png)

- fine-tuning된 CNN에서 추출한 feature를 기반으로 **클래스별 linear SVM** 학습
- softmax classifier 대신 SVM을 쓸 때 mAP가 더 높았음 (54% → 51%로 감소했으므로 SVM 선택)

## Part K — R-CNN 장점과 단점

![R-CNN 장점과 단점](images/17_rcnn_pros_cons.png)

### 장점
1. AlexNet이 Image Classification에서 보여준 것처럼 **딥러닝으로 Object Detection 성능을 크게 개선할 수 있다는 가능성을 최초로 제시**
2. Region Proposal + CNN + SVM + BBox Regression이라는 **다양한 아이디어를 체계적으로 결합**

### 단점
1. **추론 속도가 매우 느림** — GPU: 13s/image, CPU: 53s/image
   - ~2,000개 region을 각각 CNN에 통과시키기 때문
2. **파이프라인이 불필요하게 복잡** — CNN / SVM / Bounding Box LinearRegression 세 단계가 분리됨
3. **학습 프로세스도 복잡** — Pre-training → Fine-tuning → SVM 학습 → BBox Regression 학습을 별도로 진행

→ 이 단점들을 개선하기 위해 **Fast R-CNN**, **Faster R-CNN**이 순차적으로 등장

---

## 한 줄 정리

**R-CNN은 AlexNet의 classification 성공을 Detection으로 확장한 첫 시도**로, Selective Search로 region을 뽑고 CNN feature + SVM으로 분류하며 BBox regression으로 위치를 보정하는 구조다. 정확도는 높았지만 속도·복잡도 문제가 남아 Fast/Faster R-CNN으로 이어진다.

## Part L — Fast R-CNN

> Girshick, Ross. "Fast r-cnn." Proceedings of the IEEE international conference on computer vision. **ICCV 2015.**
> 논문 링크: [arxiv.org/pdf/1504.08083.pdf](https://arxiv.org/pdf/1504.08083.pdf)

![Fast R-CNN Overview](images/18_fast_rcnn_overview.png)

R-CNN의 핵심 병목인 "각 region을 독립적으로 CNN에 통과"를 제거한 구조. **이미지 전체를 ConvNet에 한 번만** 통과시켜 feature map을 만든 뒤, RoI Pooling으로 각 region의 특징을 추출한다.

---

### L-1. R-CNN Architecture의 문제점 (Slow R-CNN)

![Slow R-CNN 문제점](images/19_rcnn_slow_problem.png)

R-CNN은 ~2,000개의 region proposal 각각을 별도의 ConvNet에 통과시키기 때문에:

1. **속도가 매우 느림** — 이미지 1장 추론에 47초
2. **파이프라인이 분리** — ConvNet(feature) → SVM(분류) → BBox Regressor를 별도 학습
3. **디스크 공간 낭비** — 중간 feature를 디스크에 저장하는 구조

> R-CNN 아키텍처는 학습 파이프라인이 3단계(fine-tuning → SVM → bbox regressor)로 분리되어 있어 **end-to-end 학습이 불가능**하다는 것도 큰 한계였다.

---

### L-2. Fast R-CNN Architecture — Test time

![Fast R-CNN Architecture (Test)](images/20_fast_rcnn_architecture_test.png)

| 단계 | 설명 |
|------|------|
| ① | Input image 전체를 **ConvNet에 한 번만** 통과 → conv5 feature map 생성 |
| ② | 기존 region proposals (Selective Search)를 feature map에 **projection** |
| ③ | **RoI Pooling layer** (single-level SPP)로 각 RoI를 고정 크기로 변환 |
| ④ | Fully-connected layers → **Softmax 분류기** + **BBox Regressor** 동시 출력 |

R-CNN 대비 가장 큰 개선: ConvNet을 region 수만큼 반복하던 구조 → **단 1회 forward pass**로 대체

---

### L-3. RoI Pooling

![RoI Pooling](images/21_roi_pooling.png)

RoI Pooling은 SPP(Spatial Pyramid Pooling)의 단일 스케일 버전이다.

```
[Hi-res input] → [Convolution + Pooling] → [C×H×W feature map]
                                                   ↓
                            [RoI projection → C×h×w grid 생성]
                                                   ↓
                            [각 grid cell 내 Max-Pool → C×h×w]
                                                   ↓
                            [FC layers: 고정 크기 입력 기대]
```

- 임의 크기의 RoI를 **고정 크기(C×h×w)로 변환**하여 FC layer에 입력 가능하게 함
- 기존 R-CNN의 "crop → warp" 과정을 feature 공간에서 효율적으로 대체

---

### L-4. Fast R-CNN Architecture — Training

![Fast R-CNN Architecture (Training)](images/22_fast_rcnn_architecture_train.png)

- 전체 네트워크를 **end-to-end로 학습** 가능 (ConvNet + FC + 분류기 + bbox regressor)
- Loss: **Multi-task loss** (Log loss + Smooth L1 loss)
- 파란 화살표(gradient)가 ConvNet 전체를 통과하므로 **backbone도 함께 fine-tuning**

---

### L-5. Fast R-CNN Multi-Task Loss

![Fast R-CNN Multi-Task Loss](images/23_fast_rcnn_multitask_loss.png)

$$
L(p, u, t^u, v) = L_{\text{cls}}(p, u) + \lambda [u \geq 1] L_{\text{loc}}(t^u, v)
$$

| 항 | 설명 |
|----|------|
| $L_{\text{cls}}(p, u)$ | 분류 손실 (Log loss): 예측 확률 $p$와 정답 클래스 $u$에 대한 cross-entropy |
| $L_{\text{loc}}(t^u, v)$ | 위치 손실 (Smooth L1 loss): 예측 bbox $t^u$와 GT $v$의 차이 |
| $[u \geq 1]$ | **background (u=0)에는 위치 손실 적용 안 함** |
| $\lambda = 1$ | 모든 실험에서 동일하게 적용 |

> **Smooth L1 loss**는 L2 loss보다 이상치에 덜 민감하고 L1 loss보다 학습 안정성이 높다.

---

### L-6. Fast R-CNN Training Mini-batch

![Fast R-CNN Training Mini-batch](images/24_fast_rcnn_minibatch.png)

| 파라미터 | 값 | 설명 |
|----------|----|------|
| N (이미지 수) | **2** | 각 SGD iteration에서 랜덤하게 선택 |
| R (RoI 수) | **128** | 이미지당 64개 RoI 샘플링 |
| Positive RoI | **25% (32개)** | IoU ≥ 0.5인 GT와 겹치는 region → foreground (u ≥ 1) |
| Negative RoI | **75% (96개)** | IoU ∈ [0.1, 0.5)인 region → background (u = 0) |

> **Hierarchical sampling**: 같은 이미지에서 나온 RoI끼리 feature map을 공유하므로, 이미지 수(N)를 작게 유지하면서도 다양한 RoI를 샘플링하는 전략. R-CNN 대비 연산 공유 효율이 크게 높아진다.

### L-8. Fast R-CNN Problem

![Fast R-CNN Problem](images/26_fast_rcnn_problem.png)

**새로운 병목: Region Proposal**

R-CNN vs SPP-Net vs Fast R-CNN 비교:

- Training time: R-CNN 84h → Fast R-CNN **8.75h** (10배 단축)
- Test time (네트워크만): R-CNN 47s → Fast R-CNN **0.32s**

그런데 region proposal까지 포함하면:
- Fast R-CNN 실제 test time ≈ **2.3s** (Selective Search에 2초 소요)

> "**Runtime dominated by region proposals!**" — Selective Search가 CPU 기반으로 약 2초가 걸려 전체 inference의 bottleneck이 된다. 이를 해결하기 위해 **Faster R-CNN**에서 Region Proposal Network(RPN)을 도입한다.

---

### L-9. Fast R-CNN 장점과 단점

![Fast R-CNN 장점과 단점](images/27_fast_rcnn_pros_cons.png)

**장점**
- R-CNN의 복잡한 다단계 파이프라인 → **단일 end-to-end 학습**으로 통합
- 여러 SVM 분류기를 별도로 학습하던 방식 → **softmax + bbox regressor 동시 학습**
- 이미지당 47초 추론 → **0.32초**로 비약적 속도 향상
- 전체 네트워크가 joint training 가능 → 표현력 향상

**단점**
- Selective Search가 여전히 **CPU 기반, 이미지당 약 2초** 소요
- region proposal 포함 실제 test time은 약 **2.3초**로 real-time 불가
- Selective Search 자체가 학습 불가능한 고전 알고리즘 → gradient 역전파 불가

→ 다음 단계: **Faster R-CNN** — RPN(Region Proposal Network)으로 region proposal도 CNN 내부에서 학습

## Part M — Faster R-CNN

### M-1. Faster R-CNN 발전과정

![Faster R-CNN 발전과정](images/28_faster_rcnn_evolution.png)

> R-CNN (2014) → Fast R-CNN (2015) → **Faster R-CNN (2015)**

같은 해(2015)에 Fast R-CNN과 Faster R-CNN이 연달아 등장하며 two-stage detector의 완성형에 도달했다.

---

### M-2. Faster R-CNN 개요

![Faster R-CNN Overview](images/29_faster_rcnn_overview.png)

> Ren, Shaoqing, et al. "Faster r-cnn: Towards real-time object detection with region proposal networks." **arXiv:1506.01497 (2015).**
> 논문 링크: [arxiv.org/pdf/1506.01497.pdf](https://arxiv.org/pdf/1506.01497.pdf)

Fast R-CNN의 유일한 병목이던 **Selective Search(CPU 기반 region proposal)를 RPN(Region Proposal Network)으로 대체**한 모델. conv layer를 RPN과 Fast R-CNN이 공유하므로 거의 추가 비용 없이 region proposal이 가능하다.

- **RPN**은 "어디를 볼지 알려주는 attention" 역할
- 전체 구조: `conv layers → feature map → RPN → proposals → RoI Pooling → classifier + regressor`

---

### M-3. RPN(Region Proposal Network) Architecture

![RPN Architecture](images/30_rpn_architecture.png)

RPN은 conv feature map 위에 **sliding window**를 적용하는 작은 네트워크다.

| 구성 요소 | 설명 |
|-----------|------|
| Sliding window | conv feature map 위를 n×n 윈도우로 이동 |
| Intermediate layer | 256-d 벡터 출력 |
| cls layer | **2k scores** — 각 anchor의 objectness (object / not object) |
| reg layer | **4k coordinates** — 각 anchor의 bbox 좌표 보정 |
| k anchors | 위치당 k개의 anchor box 생성 |

**Anchor 설정:**
- 3 scales: 128², 256², 512² pixels
- 3 aspect ratios: 1:1, 1:2, 2:1
- → 위치당 **k = 9** anchors

> feature map 크기가 W×H면 총 W×H×9개의 anchor가 생성된다.

---

### M-4. RPN — Anchor Center

![RPN Anchor Center](images/32_rpn_anchor_center.png)

각 feature map grid point를 **anchor center**로 삼아 다양한 크기·비율의 anchor를 배치한다. 중심점을 기준으로 9가지 anchor box가 이미지 전 영역을 dense하게 커버한다.

![RPN Feature to Proposal](images/31_rpn_feature_to_proposal.png)

RPN의 최종 출력: feature map → **object가 있을 법한 region proposals** (bbox + objectness score)

---

### M-5. RPN Loss Function

![RPN Loss Function 1](images/33_rpn_loss_function_1.png)

$$
L(\{p_i\}, \{t_i\}) = \frac{1}{N_{cls}} \sum_i L_{cls}(p_i, p_i^*) + \lambda \frac{1}{N_{reg}} \sum_i p_i^* L_{reg}(t_i, t_i^*)
$$

| 기호 | 의미 |
|------|------|
| $i$ | mini-batch 내 anchor의 인덱스 |
| $p_i$ | anchor $i$가 object일 예측 확률 |
| $p_i^*$ | ground-truth label (positive=1, negative=0) |
| $t_i$ | 예측 bbox의 4개 파라미터 좌표 |
| $t_i^*$ | positive anchor에 대응하는 GT bbox의 4개 좌표 |

![RPN Loss Function 2](images/34_rpn_loss_function_2.png)

- $L_{cls}$: **log loss** (binary: object vs. not object)
- $L_{reg}$: **Smooth L1 loss** (Fast R-CNN의 [2]에서 정의)
- $p_i^* L_{reg}$: **positive anchor에만** regression loss 적용 ($p_i^*=0$이면 비활성화)
- $\lambda = 10$ (기본값) → cls 항과 reg 항이 roughly equally weighted

### M-6. RPN BBox Regression 파라미터화

![RPN Loss BBox Regression](images/35_rpn_loss_bbox_regression.png)

$$
t_x = \frac{x - x_a}{w_a}, \quad t_y = \frac{y - y_a}{h_a}, \quad t_w = \log\frac{w}{w_a}, \quad t_h = \log\frac{h}{h_a}
$$

$$
t_x^* = \frac{x^* - x_a}{w_a}, \quad t_y^* = \frac{y^* - y_a}{h_a}, \quad t_w^* = \log\frac{w^*}{w_a}, \quad t_h^* = \log\frac{h^*}{h_a}
$$

| 변수 | 의미 |
|------|------|
| $x, y, w, h$ | 예측 bbox의 중심좌표·폭·높이 |
| $x_a, y_a, w_a, h_a$ | anchor box의 중심좌표·폭·높이 |
| $x^*, y^*, w^*, h^*$ | ground-truth box (마찬가지로 y, w, h도 동일 패턴) |

→ "anchor box에서 nearby GT box로의 regression"으로 해석할 수 있다.

### M-7. Training RPN Mini-batch

![Training RPN Mini-batch](images/36_training_rpn_minibatch.png)

모든 anchor에 대해 loss를 최적화하면 negative sample이 압도적으로 많아 **class imbalance** 문제가 발생한다.

| 파라미터 | 값 |
|----------|----|
| Mini-batch anchor 수 | **256** |
| Positive : Negative 비율 | **최대 1:1 (128:128)** |
| Positive 수 < 128이면 | Negative로 나머지 채움 |

> Positive anchor 기준: GT box와 IoU가 가장 높은 anchor, 또는 GT box와 IoU ≥ 0.7인 anchor

### M-8. Faster R-CNN Training

![Faster R-CNN Training](images/37_faster_rcnn_training.png)

**논문 원본 방식 (Alternating training — "ugly pipeline"):**
1. ImageNet pre-trained 모델로 RPN 학습
2. RPN proposals로 Fast R-CNN 학습 (conv layer 공유 없음)
3. Fast R-CNN의 conv layer로 RPN fine-tune (conv 고정)
4. Fast R-CNN의 FC layer만 fine-tune

**현재 방식 (Joint training):**
- RPN + Fast R-CNN을 **하나의 네트워크로 동시 학습**
- **4개의 loss** 동시 최적화:
  1. RPN cls loss (anchor: good / bad)
  2. RPN reg loss (anchor → proposal bbox 보정)
  3. Fast R-CNN cls loss (proposal → class 분류)
  4. Fast R-CNN reg loss (proposal → final bbox 보정)

### M-11. Faster R-CNN 장점과 단점

![Faster R-CNN 장점과 단점](images/40_faster_rcnn_pros_cons.png)

**장점**
- Fast R-CNN의 속도 병목인 Selective Search를 **RPN으로 대체** → proposal도 GPU에서 학습 가능
- RPN과 Fast R-CNN이 conv feature를 **공유** → 추가 연산 비용 최소화
- **완전한 end-to-end 학습** 가능 (joint training)
- R-CNN 대비 **250×** 빠른 추론

**단점**
- 여전히 약 **5fps** (GPU 기준) — real-time(30fps) 미달
- Two-stage 구조 특성상 one-stage 방법(SSD, YOLO)보다 느림
- RPN + Fast R-CNN 두 모듈의 학습이 필요해 구현 복잡도가 있음

→ 이후 **one-stage detector** 계열(YOLO, SSD)과 **anchor-free** 방법들이 등장하며 속도 문제를 해결
